HTML, CSS, and JavaScript Questions

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of a webpage and consists of a series of elements which tell the browser how to display content. 
<!DOCTYPE html>
<html>
<head>
  <title>Page Title</title>
</head>
<body>
  <h1>This is a heading</h1>
  <p>This is a paragraph.</p>
</body>
</html>
Semantic elements clearly describe their meaning in a human- and machine-readable way. Examples include <article>, <section>, <nav>, <header>, and <footer>. 
<article>
  <h2>Article Title</h2>
  <p>Article content...</p>
</article>

<section>
  <h2>Section Title</h2>
  <p>Section content...</p>
</section>
Empty elements do not have any content. Examples include <br>, <hr>, and <img>. 
<br>  <!-- Line break -->
<hr>  <!-- Horizontal rule -->
<img src="image.jpg" alt="Description">  <!-- Image -->
Block elements start on a new line and take up the full width available, e.g., <div>, <p>. Inline elements do not start on a new line and only take up as much width as necessary, e.g., <span>, <a>. 
<div>This is a block element.</div>
<p>This is another block element.</p>

<span>This is an inline element.</span>
<a href="#">This is another inline element.</a>
Lists in HTML are used to group related items. Types of lists include ordered lists (<ol>), unordered lists (<ul>), and definition lists (<dl>). 
<ul>
  <li>Unordered list item</li>
  <li>Another item</li>
</ul>

<ol>
  <li>Ordered list item</li>
  <li>Another item</li>
</ol>

<dl>
  <dt>Term</dt>
  <dd>Definition of the term</dd>
</dl>
The basic structure of an HTML document includes: <!DOCTYPE html>, <html>, <head>, <title>, <meta>, <body>. 
<!DOCTYPE html>
<html>
<head>
  <title>Page Title</title>
</head>
<body>
  <!-- Content goes here -->
</body>
</html>
<!DOCTYPE html>: Defines the document type.
<html>: Root element.
<head>: Contains meta-information.
<title>: Specifies the title.
<meta>: Provides metadata.
<body>: Contains the content. 
<!DOCTYPE html>
<html>
<head>
  <title>Page Title</title>
  <meta charset="UTF-8">
</head>
<body>
  <!-- Content goes here -->
</body>
</html>
<div> is a block-level element used for structuring content, while <span> is an inline element used for styling specific parts of text. 
<div>This is a block element.</div>
<span>This is an inline element.</span>
GET appends form data to the URL, making it visible in the address bar, while POST sends form data as a request body, keeping it hidden. 
<form method="GET" action="example.com">
  <input type="text" name="username">
  <input type="submit">
</form>

<form method="POST" action="example.com">
  <input type="text" name="username">
  <input type="submit">
</form>
The <iframe> tag is used to embed another HTML document within the current document. 
<iframe src="https://www.example.com" width="600" height="400">
  <p>Your browser does not support iframes.</p>
</iframe>
localStorage allows you to store data locally within the user's browser with no expiration date. 
localStorage.setItem('key', 'value');
var value = localStorage.getItem('key');
sessionStorage allows you to store data for the duration of the page session. 
sessionStorage.setItem('key', 'value');
var value = sessionStorage.getItem('key');
localStorage persists data until explicitly deleted. sessionStorage only lasts for the duration of the page session. 
localStorage.setItem('key', 'value'); // persists
sessionStorage.setItem('key', 'value'); // lasts until tab is closed
visibility: hidden hides an element but still takes up space. display: none removes the element from the document flow. 
.hidden {
  visibility: hidden;
}

.none {
  display: none;
}
The box model in CSS consists of margins, borders, padding, and the actual content. 
element {
  width: 100px;
  padding: 10px;
  border: 5px solid black;
  margin: 20px;
}
The z-index property specifies the stack order of elements. Higher values are in front of lower values. 
.element {
  position: absolute;
  z-index: 10;
}
The ‘!important’ rule in CSS is used to override other styling rules. However, it should be used sparingly as it can make the CSS harder to maintain. 
.example {
  color: red !important;
}
Responsive designs can be created using media queries, flexible grid layouts, and responsive images. 
@media (max-width: 600px) {
  .container {
    flex-direction: column;
  }
}
The ‘box-sizing’ property in CSS defines how the width and height of an element are calculated: whether they include padding and borders or not. 
.example {
  box-sizing: border-box;
}
== checks for equality with type conversion, while === checks for equality without type conversion. 
console.log(1 == '1'); // true
console.log(1 === '1'); // false
null is an assignment value indicating no value. undefined means a variable has been declared but not assigned a value. 
let a;
console.log(a); // undefined

let b = null;
console.log(b); // null
Hoisting is JavaScript's default behavior of moving declarations to the top of the current scope. 
console.log(x); // undefined
var x = 5;

hoisted(); // Works because function declarations are hoisted
function hoisted() {
  console.log('This function is hoisted');
}
JavaScript data types include: Number, String, Boolean, Object, Undefined, Null, Symbol, and BigInt. 
let number = 42; // Number
let string = "Hello"; // String
let bool = true; // Boolean
let obj = { key: "value" }; // Object
let undef; // Undefined
let empty = null; // Null
let sym = Symbol("symbol"); // Symbol
let bigInt = 9007199254740991n; // BigInt
Promises in JavaScript represent the eventual completion (or failure) of an asynchronous operation and its resulting value. A promise is an assurance or guarantee that something will happen in the future. A person can promise another person a specific outcome or result. Promises are not limited to individuals, governments and organizations can also make promises.

You have probably made a promise before. In JavaScript, a Promise is an object that will produce a single value some time in the future.

If the promise is successful, it will produce a resolved value, but if something goes wrong then it will produce a reason why the promise failed. The possible outcomes here are similar to that of promises in real life. JavaScript promises can be in one of three possible states.

These states indicate the progress of the promise. They are: pending: This is the default state of a defined promise fulfilled: This is the state of a successful promise rejected: This is the state of a failed promise 
let promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("done!"), 1000);
});

promise.then(
  result => console.log(result), // "done!"
  error => console.log(error)
);
async and await in JavaScript are used to handle asynchronous operations in a cleaner, more readable way than using Promises alone. 
async function fetchData() {
  let response = await fetch('https://api.example.com/data');
  let data = await response.json();
  console.log(data);
}

fetchData();
A try-catch block in JavaScript is used to handle exceptions. Code within try runs, and if an error occurs, execution jumps to the catch block. 
try {
  let result = riskyOperation();
} catch (error) {
  console.log(error);
}
